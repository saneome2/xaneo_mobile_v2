### Почему AAC не работает?

Это **известная проблема ExoPlayer** (используется just_audio на Android):
- ExoPlayer не может правильно индексировать AAC файлы для seek
- Проблема существует с 2021 года
- Исправление требует изменений в ExoPlayer, не в приложении

## 🔍 Определение формата

Приложение автоматически определяет формат по:
1. Расширению файла (`.m4a` = AAC, `.ogg` = Opus)
2. MIME type из метаданных

Логи показывают:
```
⚠️ AAC format detected - seek may not work properly
💡 Recommendation: Update web client to use Opus codec
```

## 💡 Решения

### Краткосрочное (текущее):
✅ **Реализовано:**
- Мобильное приложение записывает в Opus
- UI работает для всех форматов
- Пользователь видит визуальную обратную связь
- AAC файлы воспроизводятся (без seek)

### Долгосрочное (рекомендуется):

#### 1. Обновить веб-клиент ⭐ РЕКОМЕНДУЕТСЯ
**Изменить кодек записи на веб-клиенте:**
```javascript
// Старый код (AAC):
const mediaRecorder = new MediaRecorder(stream, {
  mimeType: 'audio/mp4; codecs=mp4a.40.2'  // AAC
});

// Новый код (Opus):
const mediaRecorder = new MediaRecorder(stream, {
  mimeType: 'audio/webm; codecs=opus'  // Opus
});
```

**Преимущества:**
- Seek будет работать для всех новых записей
- Меньший размер файлов (~30% экономии)
- Лучшее качество для речи
- Совместимость с мобильным приложением

#### 2. Конвертация на сервере (опционально)
- Автоматически конвертировать AAC → Opus при загрузке
- Требует FFmpeg на сервере
- Увеличивает нагрузку на сервер

#### 3. Конвертация на клиенте (не рекомендуется)
- Конвертировать AAC → Opus на мобильном устройстве
- Требует FFmpeg plugin для Flutter
- Увеличивает размер приложения и расход батареи
